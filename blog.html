IPv4 & IPv6 Makes BGP Anomaly Detection Difficult
 The coupled_tracert program is complete and working as intended. However, I had some issues with my virtual Ubuntu machine. Evidently I cannot call traceroute using a VM. At least not without major tweaking, which is another animal. I have not explored the issue, but I assume it involves the way VirtualBox interfaces with networking protocols of the host. Regardless, I now need to run coupled_tracert from the Windows cmd. Unfortunately, we need to run coupled_tracert on 100 or more domains at a time, and that requires a simple batch script. I have never needed to use batch prior to this project, so I am thankful to learn this functionality is possible in one line: 
for /F "tokens=*" %%A in (output.txt) do coupled_tracert.py %%A
So, the script works brilliantly, but we ran into an interesting issue. Evidently only 10 of Alexa's top 100 websites actually support IPv6.. I can admit, I expected more. In hindsight, however, this makes a lot of sense. I mean, IPv6 requires hardware, and hardware requires money. Not to mention IPv6 is a solution to an address space issue, so it may be that a number of major companies exist in regions of the world that simply have not run out of IPv4 address space yet. I am actually interested in keeping output.txt and potentially running coupled_tracert again next year, just to see. Could be neat. The next step here is to find at least 100 sites that are currently supporting IPv6.

Computer & Network Security: IPv4 and IPv6 Makes BGP Anomaly Detection Difficult
 This particular project is a part of a class project for CIS 433: Computer & Network Security. The first portion of the project involves the comparison of IPv6 and IPv4 traceroute results for the first 100 of Alexa's top 500 global sites. We would like to compare results for each protocol based on hop count and hop variability. Going a step further, we intend to compare the AS (autonomous system) paths of these results. Every router encountered by our traces belongs to an AS, and we would like to learn which. For now, we need to be researching a potential solution for IP to AS resolution while, in the meantime, gathering trace data and designing a relational database model for storing said data. As of this post, we have written a couple of simple Python programs to get the ball rolling:

  • coupled_tracert.py - Takes domain name as string argument and creates a new text file of the format "domain_month_date.txt." Then, two traceroutes are sent to the domain for IPv6 and IPv4 respectively. Traceroute results are parsed and stored in the newly generated text file.
  • topsitescraper.py - A standard web scraper intended to pull the top 100 domains listed on www.alexa.com, storing them in a newly generated text file titled "output.txt."

Note, we will write a simple script to run coupled_tracert.py against each line of output.txt
